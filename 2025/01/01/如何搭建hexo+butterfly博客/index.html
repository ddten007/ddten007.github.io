博客搭建技术的选择

按网站生成类型分类可分为动态网站（Dynamic Sites）和静态网站（Static Sites）。

动态网站的特点：通过服务器端脚本实时生成页面内容，通常与数据库交互来获取数据。

常用的有：WordPress (PHP)、Joomla (PHP)、Drupal (PHP)、Ghost (Node.js)。

静态网站的特点：页面在构建时生成，部署后不会动态改变，除非重新构建。它们通常更安全、更快且易于缓存。

常用的有：Jekyll (Ruby)、Hexo (Node.js)、Hugo (Go)、Gatsby (React)、Gridsome (Vue)。

选择Hexo的原因是它属于静态网站，可以用GitHub pages服务部署网站，省掉了一个服务器，再就是Hexo基于Node.js,比较简单易用。

6. 连接 Github

推送过程中需要输入你的github用户名和密码。但是在2021年8月14日开始github官方就加强安全访问。不能通过原有账号密码git访问，密码需要用官方的token或者采用ssh公私钥访问。否则会出现下图：鉴权失败（用户名密码错误）

使用邮箱注册 GitHub 账户，选择免费账户（Free），并完成邮件验证。

右键 -> Git Bash Here，设置用户名和邮箱：

创建 SSH 密匙：

输入 ssh-keygen -t rsa -C "GitHub 邮箱"，然后一路回车。

添加密匙：

进入 [C:\Users\用户名.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。

登陆 GitHub ，进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。

Title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。

验证连接：

打开 Git Bash，输入 ssh -T git@github.com 出现 “Are you sure……”，输入 yes 回车确认。

显示 “Hi xxx! You’ve successfully……” 即连接成功。

解决方式2：就是获取token：登录github设置setting->Developer Settings->Prosonal access tokens 创建一个新token。然后就可以拿这个token当密码输入了。

用户名和token输入后，上传成功。
